# Перенос структурного знания из синтетических языков

Маленький трансформер сначала учат на **искусственном языке без смысла** (просто поток токенов по строгому правилу), затем «тело» модели замораживают и проверяют, насколько оно помогает с настоящим **английским**. Вопрос проекта: разные искусственные структуры тренируют разные умения — **какое из них реально полезно для английского?**

> Развитие статьи Budnikov & Yamshchikov, *Transfer of Structural Knowledge from Synthetic Languages* ([arXiv:2505.15769](https://arxiv.org/abs/2505.15769)).

## TL;DR

- Ввели два своих синтетических языка под конкретные умения: `counting` (счёт) и `mirror` (копирование / память).
- Меряем перенос честно: метрика с доверительными интервалами (3 сида) против сильных языков статьи `nested` (скобки) и `flat_shuffle`.
- **Главный вывод:** то, что язык сложнее устроен, не делает его полезнее для английского. Лучше всех переносится иерархия (`nested`); `mirror` идеально копирует, но для английского почти бесполезен.

## Как это работает

1. **Претрейн** — обучаем модель с нуля на синтетическом языке (учатся все веса).
2. **Перенос** — замораживаем «тело» трансформера, заново случайно инициализируем эмбеддинги под английский и дообучаем только интерфейс: либо эмбеддинги (режим **E**), либо эмбеддинги + нормализацию + последний блок (режим **ELT**).
3. **Оценка** — перплексия на английском → **Efficiency Score** = `ppl_scratch / ppl_model × 100%` (100% = модель, обученная на английском с нуля), плюс **cloze-accuracy** (30 вопросов «вставь слово», 50% = наугад).

## Синтетические языки

Каждый — это строки фиксированной длины (128 токенов) с маленьким словарём, порождённые жёстким правилом:

| Язык | Что генерирует (правило) | Какой навык |
|------|--------------------------|-------------|
| `nested` | сбалансированные вложенные скобки | иерархия |
| `flat_shuffle` | скобки + блочные перестановки (язык из статьи) | структура (сильный бейзлайн) |
| `counting` | блок «счётчик n» и ровно n одинаковых предметов | счёт |
| `mirror` | строка S, разделитель, затем S наоборот | копирование / память |

## Данные и модель

- **Английский:** TinyStories (простые детские рассказы), потоком, куски по 128 токенов. Словарь пересобран в компактный (~5900 токенов) только из встречающихся — так каждый эмбеддинг реально обучается.
- **Модель:** TinyStories-8M (GPT-Neo, скрытый размер 256, ~7 млн параметров). Веса всегда с нуля — синтетический претрейн не «видит» английский.

## Результаты

Прогон `fast_ci`, 3 сида, режим ELT, `scratch` = 59.9 ppl (= 100%):

| Язык | Efficiency (% от scratch) | cloze-acc |
|------|---------------------------|-----------|
| `nested` (статья) | **104 ± 8 %** | 58 ± 7 % |
| `flat_shuffle` (статья) | 91 ± 1 % | 64 ± 11 % |
| `counting` (новый) | 84 ± 2 % | **68 ± 4 %** |
| `mirror` (новый) | 18 ± 3 % | 58 ± 2 % |

**Главное:**

1. **Лучше всех переносится иерархия (`nested`)** — на уровне scratch и даже чуть выше. Наши новые языки её не побили: оба языка статьи выше обоих новых.
2. **`counting` уступает обоим языкам статьи, но переносится стабильно** (маленький разброс по сидам).
3. **`mirror` — далеко худший (18%)**, хотя само копирование он выучивает почти идеально и по вниманию у него самые длинные и острые головы. Это главный вывод: **вычислительная сложность языка ≠ его польза для английского**.
4. **По вниманию** `mirror` выделяется (самые дальнобойные и острые головы), но чистой «зеркальной головы» не нашлось — внимание слабое и размазанное. По cloze языки почти неразличимы.

**Практический вывод.** Разные синтетические языки качают разные, почти не пересекающиеся умения. Значит, искать один «лучший для всего» язык — неправильно; перспективнее обучать модель на **смеси** разных языков, меняя их пропорции и порядок подачи (curriculum), чтобы забрать сразу несколько полезных умений.

> **Оговорки.** Это MVP: мало шагов и всего 3 сида, числа заметно шумят между прогонами. Анализ внимания, экстраполяция по длине и изотропия считаются на одном сиде. Для финальных чисел нужен `PROFILE="convergence"` и ≥5 сидов.

## Что нового по сравнению со статьёй

- Два собственных языка под конкретные умения (`counting`, `mirror`).
- Метрика переноса с доверительными интервалами и сравнением с сильнейшим языком статьи.
- Анализ **внимания** (а не только эмбеддингов): дальнобойность и фокус голов, поиск «зеркальной головы».
- Проверка обобщения на строки **длиннее обучающих** и численные метрики изотропии эмбеддингов.

## Запуск

Проще всего — в Google Colab (бесплатный T4):

1. Открыть `synthetic_languages_mvp.ipynb` в Colab.
2. Runtime → Change runtime type → **T4 GPU**.
3. В ячейке `CONFIG` выбрать профиль:
   - `PROFILE = "quick"` — быстрый тест кода (~5–10 мин);
   - `PROFILE = "fast_ci"` — основной режим с интервалами (~1 час);
   - `PROFILE = "convergence"` — для финальных чисел (долго).
4. Runtime → **Run all**.

Зависимости (`torch`, `transformers`, `datasets`) ставятся первой ячейкой.
